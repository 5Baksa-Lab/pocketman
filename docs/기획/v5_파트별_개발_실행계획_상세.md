## 0. 목표와 성공 기준

### 최종 목표
1. 사용자 사진 업로드 후 28차원 벡터 기반 Top-3 포켓몬 매칭이 안정적으로 동작한다.
2. 사용자가 Top-3 중 1개를 선택하면 이미지/스토리/영상 Job이 생성된다.
3. 결과물 공유 및 광장 등록까지 E2E 데모가 중단 없이 진행된다.

### 최종 성공 지표 (발표 전 Day 15 기준)
| 항목 | 목표 |
|---|---|
| 매칭 API 성공률 | 95% 이상 |
| 매칭 API 응답 시간 | p95 2초 이하 |
| 이미지+스토리 생성 완료율 | 90% 이상 |
| Veo Job 상태 추적 성공률 | 90% 이상 |
| E2E 시연 시나리오 성공률 | 100% (리허설 3회 연속) |

## 1. 운영 규칙 (스파게티 코드 방지)

1. **Schema First**: 28차원 스키마는 Day 1 오전 동결, 변경은 RFC 승인 후 반영.
2. **API First**: Day 1 오후 OpenAPI 동결, 파트 간 통신은 명세 기반으로만 진행.
3. **Layer 분리**: Router-Service-Repository-Adapter 계층 강제.
4. **모듈 경계 고정**: CV 추출 / 매칭 / 생성 / SNS 경계를 분리하고 교차 의존 금지.
5. **Feature Flag**: `USE_MOCK_AI`, `USE_REAL_VEO`, `USE_REAL_SHARE`를 기본 운영 수단으로 사용.
6. **테스트 게이트**: Unit + Integration 실패 시 병합 금지.
7. **관측성 내장**: 모든 핵심 API에 `request_id`, `stage`, `duration_ms`, `error_code` 로그를 남긴다.
8. **개발일지 의무화**: 파트의 단계(Stage) 종료 시 개발일지 미작성 상태에서는 다음 Stage 착수 금지.
