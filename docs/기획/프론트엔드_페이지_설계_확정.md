## 3. 페이지별 상세 설계

### 3.1 `/intro` — 인트로

**역할**: 서비스 첫 진입 시네마틱 경험. 사용자가 클릭해야 다음 단계로 넘어감.

**레이아웃**: 풀스크린 (Header/Nav 없음), 검정 배경

**5단계 시퀀스**:
1. 어두운 배경에 스타팅 포켓몬 실루엣 등장 (랜덤 5종: 피카츄/파이리/꼬부기/이상해씨/뮤츠)
2. 스캔 라인 애니메이션 (위→아래 sweep)
3. 실루엣 → 포켓몬 이미지 reveal
4. 텍스트 등장: "당신과 닮은 포켓몬이 있습니다"
5. CTA 버튼: "시작하기 →" 클릭 시 `/` 이동

**포켓몬 이미지 소스**: `serebii.net/pokemongo/pokemon/{id}.png` (Pokemon GO 3D 아트)

**진행 방식**: 자동 진행 없음, 사용자 클릭으로만 단계 전환

### 3.2 `/` — 랜딩

**역할**: 서비스 소개. KTM RC 390 랜딩 레퍼런스 기반 스크롤 스토리텔링.

**구조** (섹션별 스크롤):

```
HERO 섹션 (다크 배경)
  - 좌: 인지도 높은 포켓몬 이미지 (피카츄 등 스타팅)
  - 우: 실제 샘플 얼굴 사진 (Unsplash)
  - Ghost text 레이어 기법 적용

ANALYSIS 섹션
  - "얼굴을 분석합니다" + 시각적 설명

MATCHING 섹션
  - "386마리 포켓몬과 비교합니다" + 시각적 설명

CREATION 섹션
  - "나만의 크리처가 탄생합니다" + 시각적 설명

CTA 섹션
  - "지금 시작하기" 버튼 → /upload
```

**Ghost text 기법**: 배경에 반투명 대형 텍스트 레이어 + 전경 콘텐츠 오버레이

**Intersection Observer**: 스크롤 진입 시 애니메이션 트리거

### 3.3 `/login` — 로그인

**역할**: 신규/기존 사용자 진입점. 인증 후 `?next=` 파라미터 경로 or `/upload`로 이동.

#### 데스크톱/태블릿 (Instagram 로그인 구조 적용)

- 좌우 50/50 분할
- **좌측**: Pocketman 브랜딩 + 포켓몬 크리처 콜라주
  - 피카츄/파이리/뮤츠 카드 3장 부채꼴 배치 (각도 -8deg, +5deg, -3deg)
  - 각 포켓몬 눈 애니메이션 (좌우 이동 + 깜빡임, 개별 딜레이)
  - 배경: beige → coral/teal 오로라 그라데이언트
- **우측**: 로그인 폼 (흰 배경, 수직 중앙 정렬)
  - 소셜 로그인 우선 노출 (Google → Kakao 순)
  - 이메일/비밀번호 폼
  - 비밀번호 찾기 / 회원가입 링크

#### 모바일 (Dribbble 레퍼런스 적용)
- 참조: https://dribbble.com/shots/6670990-Login-screen-animated-illustration
- 상단 55%: 포켓몬 마을(팔레트 타운) 일러스트 애니메이션
  - 배경: 하늘색(teal 계열)
  - 요소: Professor Oak 연구소 + 주인공 집 + 나무 + 도로
  - 애니메이션: 구름 이동(8s) + Pidgey 비행(6s) + 피카츄 꼬리(0.5s) + 풀 흔들림(2s)
  - 색상: `#2DA2A0`→`#009087`, `#1B584E`, `#4A992D`, `#C8A43E`
- 하단 45%: 흰 카드 슬라이드업 + 로그인 폼

#### 눈 애니메이션 CSS

```css
@keyframes eyes-look {
  0%, 45%  { transform: translateX(0); }
  50%, 65% { transform: translateX(2px); }
  70%, 85% { transform: translateX(-2px); }
  90%, 100%{ transform: translateX(0); }
}
@keyframes eye-blink {
  0%, 96%  { transform: scaleY(1); }
  97%, 99% { transform: scaleY(0.1); }
  100%     { transform: scaleY(1); }
}
```

#### 인증 기술 스택

- **NextAuth.js (Auth.js v5)** 사용
- `session: { strategy: "jwt" }` — JWT 방식
- 지원 Provider: Google, Kakao, Credentials (이메일)
- Access Token: 15분, Refresh Token: 7일 (httpOnly cookie)
- Route 보호: `middleware.ts`

### 3.4 `/signup` — 회원가입

**레이아웃**: 로그인 페이지와 동일 구조 (일관성 유지)

**폼 필드**:
- 이름 (2~20자)
- 이메일 (RFC 형식)
- 비밀번호 (8자 이상, 영문+숫자, 강도 인디케이터)

**정책**:
- 이메일 인증 생략 (MVP)
- 가입 즉시 자동 로그인 → `/upload`
- 중복 이메일: 서버 409 → "이미 사용 중인 이메일입니다"

**소셜 가입 플로우**:
```
signIn("google") → OAuth → 백엔드 /auth/social → JWT → 세션
```

### 3.5 `/upload` — 업로드

**역할**: 사용자가 핵심 가치를 처음 경험하는 페이지. 불안 요소 최소화.

**레이아웃**:
- 헤드라인: "당신과 닮은 포켓몬을 찾아드립니다"
- 드롭존: 점선 테두리 → 드래그 오버 시 코랄 테두리
- "샘플 사진으로 체험하기" (ghost 버튼) — 비로그인도 가능
- "🔒 사진은 분석 후 즉시 삭제됩니다" 신뢰 문구

**상태별 드롭존**:
| 상태 | UI |
|------|-----|
| 기본 | 점선 테두리, 회색 아이콘 |
| 드래그 오버 | 코랄 테두리 + 배경 틴트 |
| 파일 선택됨 | 미리보기 썸네일 + "분석 시작" 버튼 |
| 분석 중 | Spinner + "포켓몬을 찾는 중..." |
| 에러 | 빨간 테두리 + 인라인 에러 |

**검증**: MIME 타입 + 10MB 제한 + 100×100px 최소 해상도

**소프트 게이트**: 비로그인 사용자도 체험 가능 → 결과 저장 시 로그인 유도

### 3.6 `/match` — 매칭 결과

**역할**: 분석 결과 TOP-3 포켓몬 표시 → 하나 선택 → 크리처 생성 시작.

**포켓몬 이미지 소스**: `serebii.net/pokemongo/pokemon/{id}.png` (Pokemon GO 3D 아트)

**카드 디자인**: davidkpiano CodePen (https://codepen.io/davidkpiano/pen/NreaMB) 구조 그대로 적용

**카드 레이아웃**:
```
배경: 포켓몬 타입별 색상 (불꽃=#c97c4a, 물=#5b8fc9, 전기=#c9a93b 등)
     + 다이아몬드 패턴 (repeating-linear-gradient)

좌측: Pokemon GO 아트 이미지 + 라디얼 글로우 + head 바운스 애니메이션
우측: ◉ {번호}  {이름} (Lato 폰트, 자간 넓게)
      Type: {타입}
      유사도: {%}
      {포켓몬 카테고리명}
      {PokeAPI 설명 텍스트}
      유사도 프로그레스바
      [이 포켓몬으로 시작하기] 버튼
```

**배치**: TOP-3 카드 가로 3열, 1위 카드 중앙 + scale-105 + "BEST MATCH" 뱃지

**선택 시**: 선택 카드 scale-up + 나머지 fade-out → 0.6s 후 `/generate/[id]`

**데이터**: `sessionStorage` (MatchResultStorage) → 없으면 `/upload` 리다이렉트

### 3.7 `/generate/[id]` — 생성 대기

**역할**: 크리처 생성 중 대기. 기다림을 흥미로운 경험으로 전환.

**디자인**: simeydotme CodePen (https://codepen.io/simeydotme/pen/ZGzrBQ) 포켓몬 퍼레이드 애니메이션 적용

**레이아웃** (풀스크린, Header 없음):
```
배경: #1D1F20 (다크)
상단: Pocketman 로고 (opacity 0.8)
중앙: 생성 상태 텍스트 (흰색, opacity 0.5)
하단: 포켓몬 퍼레이드 애니메이션 (스프라이트 시트 기반)
```

**상태 텍스트 순서**:
- 0s~: "나만의 크리처가 태어나고 있어요..."
- 10s~: "포켓몬의 특성을 융합하고 있어요..."
- 25s~: "거의 다 됐어요! 조금만 기다려주세요"

**원본 대비 변경사항**:
- `document.body.onclick` 트리거 제거 → `useEffect` 마운트 즉시 실행
- Pokemon 로고 → Pocketman 로고 컴포넌트
- "Click Anywhere to Begin" 텍스트 제거
- Veo 폴링 (`useVeoPolling`, 3초 간격, 최대 25회) 병렬 실행
- 완료 시 `/result/[id]` 자동 이동

**스프라이트 소스**: `https://assets.codepen.io/13471/pokemon-sprite.png`

### 3.8 `/result/[id]` — 크리처 결과

**역할**: 서비스 클라이맥스. 크리처 첫 공개 + 공유 유도.

**디자인**: davidkpiano 카드 레이아웃 그대로 적용 (match 페이지와 동일 구조)

**레이아웃**:
```
배경: 크리처 주 타입 색상 + 다이아몬드 패턴
좌측: 크리처 영상/이미지 + 라디얼 글로우 + 🎉 컨페티 파티클
우측: ◉ {번호}  {크리처 이름} (인라인 편집 가능)
      Type: {타입 배지들}
      닮은 포켓몬: {포켓몬명} {유사도}%
      {크리처 설명}
      [📤 공유] [🌍 광장에 올리기]
      [💾 저장] [🔄 다시 만들기]
```

**등장 애니메이션 시퀀스**:
1. 0.0s: 배경 페이드인
2. 0.3s: 카드 scale(0.8→1.0) + fadeIn
3. 0.8s: 컨페티 파티클 폭발 (코랄+틸+골드)
4. 1.2s: 이름 + 타입 배지 슬라이드업
5. 1.8s: 공유 버튼 그룹 fadeIn

**이름 인라인 편집**: 탭 → input 전환 → Enter/✅ → `PATCH /creatures/{id}/name`

**크리처 영상**: 클릭 시 재생 (자동재생 없음)

**비로그인 처리**: 저장/공유 시 로그인 유도 모달 → 로그인 후 동작 이어서 실행

### 3.9 `/plaza` — 광장

**역할**: 포켓몬 Platinum Wi-Fi 광장 스타일 실시간 인터랙티브 소셜 공간.

**기술 스택**:
- 렌더링: Phaser.js (타일맵 + 스프라이트 + 카메라 내장)
- 실시간: socket.io (WebSocket)
- 맵: 32×32 타일, 타일 1개 = 32px (1024×1024 월드)
- 아트 스타일: 픽셀아트 탑뷰

**맵 구성**:
- 바닥: 체크무늬 타일 (베이지/연한 틸)
- 중앙: 분수 (랜드마크)
- 장식: 나무/덤불 (이동 불가 영역)
- 건물: 장식용 픽셀아트

**크리처 스프라이트**:
```
원형 마스크 크리처 이미지 (32×32px 또는 48×48px)
+ 픽셀 테두리
+ 닉네임 태그 (말풍선 스타일)
이동 애니메이션: 2프레임 bounce
정지 애니메이션: idle breathing (scale 1.0 → 1.03)
```

**HUD 구성**:
- 상단: [← 나가기] / 온라인 인원 / [🔇 BGM] / [채팅]
- 하단 좌: D-pad (모바일)
- 하단 우: 온라인 유저 목록

**타 플레이어 클릭**: davidkpiano 카드 디자인 모달 팝업

#### 공개 채팅 (말풍선)

- [채팅] 버튼 → 입력창 활성화 (모바일: 키패드 올라옴)
- 최대 20자 입력 → 전송
- 크리처 머리 위 말풍선 3초 표시 후 fadeOut
- WebSocket broadcast → 전체 유저 동시 표시

**말풍선 디자인**: 흰 배경 + 2px 픽셀 테두리 + 꼬리

#### 비밀 채팅 (1:1 DM)

- 타 플레이어 상세 카드 → [비밀채팅] 버튼
- 상대방에게 수락 요청 팝업
- 수락 → 양쪽 채팅창 오픈 (화면 우하단 float)
- 거절 → "거절됐어요" Toast
- 채팅창 닫기 → 대화 종료 (히스토리 미저장, MVP)
- 최대 20자 제한

#### 크리처 없는 신규 유저

진입 시 안내 모달:
```
광장을 이용하려면 먼저 크리처를 만들어야 해요!
[지금 크리처 만들러 가기]
```

### 3.10 `/creatures/[id]` — 크리처 상세

**역할**: 광장/링크 등에서 진입하는 공개 크리처 상세 페이지. SNS 게시물 상세 역할.

**레이아웃**:
- davidkpiano 카드 디자인 (match/result 페이지와 동일)
- 하단: 댓글 섹션

**좌측**: 크리처 영상/이미지 (클릭 시 재생, 자동재생 없음)

**우측 정보**:
```
◉ {번호}  {크리처 이름}
Type: {타입 배지들}
by @{username}
{날짜}

닮은 포켓몬: {포켓몬명} {유사도}%
{크리처 설명}

[버튼 - 내/타인 분기]
```

**버튼 분기**:
| 상황 | 버튼 |
|------|------|
| 내 크리처 | [이름 편집] [공개/비공개 전환] [삭제] [공유] |
| 타인 크리처 | [❤️ 좋아요] [📤 공유] [광장에서 찾기] |

**광장에서 찾기**:
- 소유자 광장 접속 중: 활성화 → 클릭 시 `/plaza` + 해당 크리처 위치 포커스
- 오프라인: 비활성화 (회색)

**댓글**:
- 최신순 정렬
- 최대 100자
- 본인 댓글만 삭제 가능
- 비로그인: 댓글창 탭 시 로그인 유도

### 3.11 `/my` — 마이페이지

**역할**: 개인 계정 + 크리처 관리 허브. Instagram 프로필 구조 기반.

**레이아웃**:
```
프로필 헤더 (아바타 + 닉네임 + 한줄소개 + 통계)
[프로필 편집] 버튼

탭: [내 크리처] / [좋아요한 크리처]
크리처 그리드 (공개🌍 / 비공개🔒 표시)

설정 섹션
계정 섹션
```

**아바타**: 대표 크리처 이미지 (원형)

#### `/my/edit` — 프로필 편집

- 상단: 실시간 미리보기 카드 (davidkpiano 스타일, 편집 내용 즉시 반영)
- 닉네임 (중복 확인 필수)
- 한 줄 소개 (최대 50자)
- 대표 크리처 선택 (가로 스크롤 목록)

**닉네임 중복 검사**: `GET /users/check-nickname?q={nickname}` → 실시간 확인

#### `/my/password` — 비밀번호 변경

- 현재 비밀번호 확인
- 새 비밀번호 (8자 이상, 강도 인디케이터)
- 새 비밀번호 확인
- 소셜 로그인 전용 계정: 해당 섹션 숨김

#### 설정

**다크 모드**:
- 토글 스위치
- 저장: `localStorage` + DB 동기화
- 기본값: 시스템 설정 (`prefers-color-scheme`) 따름
- 전환: `transition: background-color 0.3s ease`

**글자 크기 조절**:
- 3단계: 작게(14px) / 보통(16px, 기본) / 크게(18px)
- 슬라이더 조작 시 즉시 미리보기 적용
- `html { font-size }` 변경으로 전체 적용

#### 회원 탈퇴

- 2단계 확인 (경고 모달 → 비밀번호/텍스트 확인)
- 탈퇴 즉시 크리처 + 댓글 + 좋아요 기록 + 계정 정보 전체 삭제
- 복구 불가

## 5. 인증 설계

### DB (Railway PostgreSQL)

```sql
CREATE TABLE users (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  email       VARCHAR(255) UNIQUE NOT NULL,
  name        VARCHAR(100) NOT NULL,
  password    VARCHAR(255),           -- 소셜 가입 시 NULL
  provider    VARCHAR(20),            -- 'google' | 'kakao' | null
  provider_id VARCHAR(255),
  avatar_url  VARCHAR(500),
  bio         VARCHAR(100),           -- 한 줄 소개
  created_at  TIMESTAMPTZ DEFAULT NOW()
);
```

### 백엔드 엔드포인트 (추가 필요)

```
POST /auth/register    { name, email, password }
POST /auth/login       { email, password } → { access_token }
POST /auth/social      { provider, code } → { access_token }
GET  /auth/me          → { id, name, email, bio, avatar_url }
GET  /users/check-nickname?q={nickname}
PATCH /users/me        { name, bio }
PATCH /users/me/password
DELETE /users/me
```

### Route 보호 (middleware.ts)

보호 대상: `/upload`, `/generate/*`, `/result/*`, `/my/*`, `/plaza` (크리처 없으면 안내)
