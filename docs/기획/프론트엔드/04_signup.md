# 페이지 설계: `/signup` — 회원가입

- 라우트: `/signup`
- 상태: 설계 확정 (미구현)

---

## 역할

신규 사용자 이메일 계정 생성. 가입 즉시 자동 로그인 후 `/upload`로 이동.

---

## 레이아웃

로그인 페이지(`/login`)와 동일 구조 (일관성 유지).
- 데스크톱/태블릿: 좌우 50/50 분할 (좌: 브랜딩 패널, 우: 회원가입 폼)
- 모바일: 팔레트 타운 일러스트 상단 + 폼 하단

---

## 폼 필드

| 필드 | 유효성 검사 |
|------|------------|
| 이름 | 2~20자 |
| 이메일 | RFC 5321 형식 |
| 비밀번호 | 8자 이상, 영문+숫자 혼합 |
| 비밀번호 확인 | 위와 동일 여부 |

**비밀번호 강도 인디케이터**: 약함 / 보통 / 강함 (색상 바)

---

## 정책

- 이메일 인증 생략 (MVP)
- 가입 즉시 자동 로그인 → `/upload`로 이동
- 중복 이메일 → 서버 409 응답 → "이미 사용 중인 이메일입니다" 인라인 에러

---

## 소셜 가입 플로우

```
[Google로 가입] 클릭
  → OAuth 동의 화면
  → 백엔드 POST /auth/social { provider: "google", code }
  → JWT 발급
  → NextAuth 세션 생성
  → /upload 이동
```

소셜 가입 시 비밀번호 없음 → `/my/password` 페이지에서 비밀번호 섹션 숨김.

---

## API 연동

- `POST /auth/register` → `{ name, email, password }`
- 성공 시 NextAuth `signIn("credentials")` 자동 호출
