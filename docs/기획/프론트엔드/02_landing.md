# 페이지 설계: `/` — 랜딩

- 라우트: `/`
- 상태: 설계 확정 (F1에서 기본 구현 완료, 고도화 예정)

---

## 역할

서비스 소개. 스크롤 스토리텔링으로 서비스 가치 전달 → 업로드 CTA 유도.

---

## 레이아웃 (섹션별 스크롤)

```
HERO 섹션 (다크 배경)
  - 좌: 인지도 높은 포켓몬 이미지 (피카츄 등 스타팅 5종)
  - 우: 실제 샘플 얼굴 사진 (Unsplash)
  - Ghost text 레이어 기법 적용

ANALYSIS 섹션
  - "얼굴을 분석합니다" + 시각적 설명

MATCHING 섹션
  - "386마리 포켓몬과 비교합니다" + 시각적 설명

CREATION 섹션
  - "나만의 크리처가 탄생합니다" + 시각적 설명

CTA 섹션
  - "지금 시작하기" 버튼 → /upload
```

---

## Ghost Text 기법

배경에 반투명 대형 텍스트 레이어 + 전경 콘텐츠 오버레이.
예: 배경에 "POKEMON" 대문자 (opacity 0.05) + 위에 실제 콘텐츠 배치.

---

## 인터랙션

- **Intersection Observer**: 각 섹션 진입 시 애니메이션 트리거 (fadeIn, slideUp)
- 스크롤 속도에 따른 parallax 효과 (선택적)

---

## 샘플 크리처 섹션

- 최하단에 공개된 크리처 6개 그리드 표시
- API 오류 시 섹션 숨김 (silent fail)
- Skeleton loader 추가 예정 (이슈로 등록)
