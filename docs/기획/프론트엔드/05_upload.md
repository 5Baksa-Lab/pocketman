# 페이지 설계: `/upload` — 업로드

- 라우트: `/upload`
- 상태: 설계 확정 (F1에서 구현 완료)

---

## 역할

사용자가 핵심 가치를 처음 경험하는 진입점. 불안 요소 최소화, 신뢰 확보.

---

## 레이아웃

- 헤드라인: "당신과 닮은 포켓몬을 찾아드립니다"
- 드롭존: 중앙 배치, 점선 테두리
- "샘플 사진으로 체험하기" ghost 버튼 (비로그인도 사용 가능)
- "🔒 사진은 분석 후 즉시 삭제됩니다" 신뢰 문구

---

## 드롭존 상태별 UI

| 상태 | UI |
|------|-----|
| 기본 | 점선 테두리, 회색 아이콘 |
| 드래그 오버 | 코랄(`#f15946`) 테두리 + 배경 틴트 |
| 파일 선택됨 | 미리보기 썸네일 + "분석 시작" 버튼 |
| 분석 중 | Spinner + "포켓몬을 찾는 중..." |
| 에러 | 빨간 테두리 + 인라인 에러 메시지 |

---

## 파일 검증

- MIME 타입: `image/jpeg`, `image/png`, `image/webp`
- 최대 크기: 10MB
- 최소 해상도: 100×100px

---

## 소프트 게이트 (비로그인 처리)

- 비로그인 사용자도 업로드 및 매칭 결과 조회 가능
- `/result/[id]` 에서 저장/공유 시 로그인 유도 모달 표시
- 로그인 완료 후 이전 동작 이어서 실행

---

## 데이터 흐름

```
파일 선택/드롭
  → 검증 통과
  → POST /match/face (multipart/form-data)
  → 응답: { matches: [{ pokemon_id, name, score, ... }] }
  → MatchResultStorage.save(matches) → sessionStorage
  → router.push('/match')
```
