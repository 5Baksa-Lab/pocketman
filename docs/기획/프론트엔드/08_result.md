# 페이지 설계: `/result/[id]` — 크리처 결과

- 라우트: `/result/[id]`
- 상태: 설계 확정 (F2에서 구현 예정)

---

## 역할

서비스 클라이맥스. 생성된 크리처 첫 공개 + 감동 연출 + 공유 유도.

---

## 카드 디자인

레퍼런스: davidkpiano CodePen `NreaMB` (match 페이지와 동일 구조)

```
배경: 크리처 주 타입 색상 + 다이아몬드 패턴

좌측:
  - 크리처 영상/이미지
  - 라디얼 글로우 효과
  - 🎉 컨페티 파티클 (canvas-confetti)

우측 (Lato 폰트):
  - ◉ #{번호}  {크리처 이름} (인라인 편집 가능)
  - Type: {타입 배지들}
  - 닮은 포켓몬: {포켓몬명} {유사도}%
  - {크리처 설명}
  - [📤 공유] [🌍 광장에 올리기]
  - [💾 저장] [🔄 다시 만들기]
```

---

## 등장 애니메이션 시퀀스

| 시간 | 동작 |
|------|------|
| 0.0s | 배경 페이드인 |
| 0.3s | 카드 scale(0.8→1.0) + fadeIn |
| 0.8s | 컨페티 파티클 폭발 (코랄 + 틸 + 골드) |
| 1.2s | 이름 + 타입 배지 슬라이드업 |
| 1.8s | 공유 버튼 그룹 fadeIn |

---

## 인라인 이름 편집

```
이름 클릭 → <input> 전환 (포커스)
Enter 또는 ✅ 클릭
  → PATCH /creatures/{id}/name { name: string }
  → 성공: 이름 업데이트 + 텍스트 모드 복귀
  → 실패: Toast 에러 + 원래 이름으로 복귀
```

---

## 크리처 미디어

- 기본: 이미지 표시
- 영상 있는 경우: 클릭 시 재생 (자동재생 없음)
- 재생 중: 좌측 패널 전체를 video 태그로 대체

---

## 버튼 동작

| 버튼 | 동작 |
|------|------|
| 📤 공유 | Web Share API → fallback: 링크 복사 |
| 🌍 광장에 올리기 | 크리처 공개 설정 → `/plaza` 이동 |
| 💾 저장 | 이미지/GIF 다운로드 |
| 🔄 다시 만들기 | `/upload`로 이동 |

---

## 비로그인 처리

- 저장/공유/광장 버튼 클릭 시 로그인 유도 모달
- 로그인 완료 후 해당 동작 자동 재실행

---

## API 연동

- `GET /creatures/{id}` → 크리처 상세 데이터
- `PATCH /creatures/{id}/name` → `{ name: string }`
