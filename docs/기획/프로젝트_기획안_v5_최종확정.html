01 프로젝트 비전

"내 얼굴을 분석해 나와 가장 닮은 포켓몬을 찾고,
그 포켓몬 스타일의 오리지널 크리처로 세상과 소통하는 익명 힐링 SNS"
구분내용
배경외모 평가에 지치거나 온라인 소통에 두려움을 느끼는 현대인에게 안전한 디지털 페르소나가 필요함
솔루션CV로 얼굴 특징 추출 → pgvector 포켓몬 DB 유사도 매칭 → Top-3 제시 → 사용자 선택 → 오리지널 크리처 + 영상 생성
타겟MZ세대, 포켓몬 팬, SNS 콘텐츠 소비자
차별점단순 결과 제공이 아닌 Top-3 근거 제시 + 사용자 직접 선택으로 참여감 강화

03 MLOps 파이프라인 시스템 아키텍처

3-1. 사전 준비 파이프라인 (배포 전 1회성 배치 작업)

1
PokeAPI 배치 수집
#001~#386 총 386마리 구조 데이터 자동 수집
수집 항목: name_kr/en, types, stats, height, weight, sprite_url, color, shape, habitat
딜레이: 100ms/호출 (rate limit 방지)
2
한국어 도감 설명 수집
PokeAPI flavor_text_entries에서 'ko' 언어 필터링
누락 시 Pokémon Fandom 위키 스크래핑으로 보완
3
Gemini Vision 시각 특징 주석 배치
입력: 포켓몬 공식 스프라이트 이미지 URL (PokeAPI)
출력: visual 10차원 JSON (eye_size, face_roundness 등)
비용: 386마리 × ~$0.001 = 약 $0.4 (1회성)
→ 팀 전체 검토 후 이상값 수동 수정
4
Gemini Flash 인상/성격 점수 추출 배치
입력: 도감 텍스트 + 타입 정보
출력: impression 9차원 JSON (cute, calm, smart, fierce 등)
5
타입 친화도 자동 계산
TYPE_TO_AFFINITY 룰 테이블 기반 자동 계산
출력: type_affinity 8차원 (water, fire, grass, electric, psychic, normal, fighting, ghost)
6
28차원 벡터 생성 및 pgvector 저장
visual(10) + impression(9) + type_affinity(8) + glasses(1) = 28차원
PostgreSQL + pgvector extension 저장
IVFFlat 코사인 유사도 인덱스 생성
7
전체 검증 및 튜닝
20+ 테스트 케이스 매칭 결과 팀 전체 리뷰
이상한 매칭 발견 시 가중치 조정 후 재확인

04 핵심 아키텍처 변경: Ollama 제거 → pgvector 임베딩

v4 방식 — Ollama 홈 서버

  • 사용자 이미지 입력
  • MediaPipe 수치 추출
  • 홈 GPU 서버에 Ollama 요청 (3~10초)
  • 멀티모달 LLM 분석
  • Rule-based 속성 매핑
  • 단일 결과 반환 (선택 불가)

v5 방식 — pgvector 임베딩 DB

  • 사용자 이미지 입력
  • MediaPipe 수치 추출
  • 공통 스키마 JSON 생성 (28차원)
  • pgvector 코사인 유사도 검색 (<500ms)
  • Top-3 포켓몬 + 근거 반환
  • 사용자가 1~3위 중 직접 선택
평가 항목Ollama 방식 (v4)pgvector 방식 (v5)승자
서버 의존성홈 GPU 서버 필수없음 (DB만 필요)v5
응답 속도3~10초 (GPU 의존)< 500msv5
월 운영 비용서버 전기세 + 유지비0원 (DB 구축 후 고정)v5
결과 일관성매 요청마다 다를 수 있음동일 입력 = 동일 결과v5
근거 제공LLM 생성 (불안정)스키마 값 기반 (명확)v5
사용자 경험단일 결과 (선택 불가)Top-3 중 직접 선택v5
