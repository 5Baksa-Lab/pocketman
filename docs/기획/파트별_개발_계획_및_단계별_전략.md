# Pokéman 파트별 개발 계획 및 단계별 전략 (Stage Plan) - 상세판

**작성일:** 2026년 3월 6일
**기반 문서:** 프로젝트 기획안 v5 (최종 확정)
**전략 목표:** 3주(15일) 동안 프론트엔드, 백엔드, MLOps 각 파트가 서로 병목 없이 독립적으로 개발을 진행하되, 각 Stage의 끝에서 완벽하게 통합되도록 구체적인 태스크와 산출물을 정의함.

---

## 1. MLOps & Data 파트 (가장 먼저 시작되어야 함)
**핵심 목표:** 386마리의 포켓몬 DB를 28차원 벡터로 완벽하게 구축하고, FastAPI에 CV 랜드마크 추출 모듈을 이식.

### Stage 1: 데이터 파이프라인 구축 (Day 1~3)
*   **[Task 1.1] PokeAPI 수집 스크립트 작성 (Python + requests)**
    *   **대상:** 1~3세대 포켓몬 386마리.
    *   **수집 항목:** `id`, `name(kr/en)`, `types`, `stats`, `sprites(official-artwork url)`.
    *   **산출물:** `pokemon_raw_data.json`
*   **[Task 1.2] 한국어 도감 텍스트 보완 스크래퍼 (BeautifulSoup/Selenium)**
    *   **대상:** PokeAPI의 `flavor_text_entries`에서 `language.name == 'ko'` 추출.
    *   **예외 처리:** 누락된 세대는 나무위키(포켓몬 목록)에서 스크래핑.
    *   **산출물:** `pokemon_pokedex_kr.json`
*   **[Task 1.3] 벡터 DB (pgvector) 초기 세팅**
    *   **작업:** PostgreSQL 인스턴스에 `CREATE EXTENSION vector;` 실행.
    *   **스키마 생성:** `id`, `name`, `vector(28)`, `metadata(JSONB)`.

### Stage 2: AI 주석 파이프라인 및 벡터화 (Day 4~6)
*   **[Task 2.1] Gemini Vision 스크립트 (이미지 분석)**
    *   **작업:** 386개의 스프라이트 이미지를 Gemini API에 전달.
    *   **프롬프트:** "이 포켓몬의 눈 크기, 미간, 둥근 정도 등 10가지 시각적 특징을 0.0~1.0 사이의 JSON 수치로 반환해."
    *   **산출물:** `visual_features_10d.json`
*   **[Task 2.2] Gemini Flash 스크립트 (텍스트 분석)**
    *   **작업:** 도감 텍스트와 속성을 기반으로 '귀여움, 사나움 등 9가지 인상 수치' 추출.
    *   **산출물:** `impression_features_9d.json`
*   **[Task 2.3] 28차원 통합 및 DB 삽입 (Python + psycopg2)**
    *   **작업:** 시각(10) + 인상(9) + 타입(8) + 기타(1) 병합 후 정규화.
    *   **DB 쿼리:** 정규화된 28차원 배열을 `INSERT INTO ...` 실행 및 코사인 인덱스(`IVFFlat`) 적용.

### Stage 3: 실시간 CV 랜드마크 추출 모듈 개발 (Day 7~9)
*   **[Task 3.1] MediaPipe Face Mesh 함수화**
    *   **작업:** `def extract_face_landmarks(image_bytes) -> dict:` 작성.
    *   **상세:** 입력된 이미지에서 468개 랜드마크 (x, y, z) 좌표 추출.
*   **[Task 3.2] 랜드마크 $\rightarrow$ 28차원 벡터 변환 로직 (가장 중요)**
    *   **작업:** 눈 가로/세로 길이 계산 (유클리디안 거리), 턱선 각도 계산 등 수학적 룰 기반 함수 작성.
    *   **산출물:** `def calculate_28d_vector(landmarks) -> list[float]:` 모듈. (백엔드 팀에 `.py` 파일로 전달).

## 2. Backend (FastAPI) 파트
**핵심 목표:** 프론트엔드의 요청을 받아 CV 분석 $\rightarrow$ DB 매칭 $\rightarrow$ 생성형 API 연동을 오케스트레이션.

### Stage 1: 코어 API 및 인프라 세팅 (Day 1~4)
*   **[Task 1.1] FastAPI 프로젝트 스캐폴딩**
    *   **작업:** Dockerfile 작성, Pydantic 모델(요청/응답 DTO) 정의.
*   **[Task 1.2] Mock 매칭 API 개발 (`POST /api/v1/match`)**
    *   **입력:** `multipart/form-data` (사진).
    *   **임시 로직:** 사진 분석 없이 DB에서 랜덤하게 3마리를 뽑아 Top-3 JSON 포맷으로 반환. (프론트 팀 작업용)
*   **[Task 1.3] 실제 pgvector 검색 쿼리 구현**
    *   **작업:** `SELECT * FROM pokemon ORDER BY vector <=> '[...]' LIMIT 3;` 쿼리를 ORM(SQLAlchemy)으로 구현.

### Stage 2: CV 통합 및 생성형 API 병렬 호출 (Day 5~9)
*   **[Task 2.1] MLOps CV 모듈 이식**
    *   **작업:** `/upload` 엔드포인트에 `calculate_28d_vector()`를 연결하여, 실제 업로드된 사진에서 벡터를 뽑아 Top-3를 찾도록 완성.
*   **[Task 2.2] Imagen 3 & Gemini Flash 연동 (`POST /api/v1/generate`)**
    *   **입력:** 유저가 선택한 `pokemon_id`.
    *   **작업:** `asyncio.gather()`를 활용해 Imagen 3(이미지 생성)와 Gemini Flash(스토리/이름)를 동시에 비동기로 호출하여 지연 시간(Latency) 최소화.
*   **[Task 2.3] S3 업로드 연동**
    *   **작업:** `boto3`를 사용해 생성된 이미지를 S3 버킷에 업로드하고, 퍼블릭 URL(Cloudflare R2 등)을 반환.

### Stage 3: 비동기 영상 생성 및 SNS API (Day 10~14)
*   **[Task 3.1] Veo API 연동 및 상태 폴링 (`POST /api/v1/video/start`, `GET /api/v1/video/status`)**
    *   **작업:** 비동기 Job을 등록하고 Job_ID 반환. 상태 폴링 엔드포인트 구현 (PENDING, PROCESSING, COMPLETED).
*   **[Task 3.2] 포켓 도감 API 구현**
    *   **작업:** `GET /api/v1/feed` (페이지네이션 적용), `POST /api/v1/react` (이모지 카운트 증가).

## 3. Frontend (Next.js) 파트
**핵심 목표:** 대기 시간을 감추는 인터랙티브 UX/UI 구현. API 지연을 방어하는 화면 설계.

### Stage 1: UI 퍼블리싱 및 Mock 연동 (Day 1~5)
*   **[Task 1.1] Next.js 14 (App Router) + Tailwind CSS 환경 세팅**
    *   **작업:** 컬러 팔레트, 폰트(도트 폰트 등), 공통 레이아웃 컴포넌트 구성.
*   **[Task 1.2] 사진 업로드 UX 개발**
    *   **작업:** Dropzone 컴포넌트, 이미지 리사이징(Canvas API 사용, 서버 부하 감소용 클라이언트 리사이즈), 로딩 스피너.
*   **[Task 1.3] Top-3 선택 화면 UI**
    *   **작업:** 백엔드의 Mock API를 받아, 3개의 카드 UI를 가로 스크롤(Swiper) 형태로 렌더링. 선택(Radio) 상태 관리.

### Stage 2: 렌더링 연출 (스토리북) 고도화 (Day 6~10)
*   **[Task 2.1] 점진적 결과 렌더링 (Optimistic UI)**
    *   **작업:** `/generate` 호출 후 텍스트(이름/스토리)가 먼저 오면 바로 화면에 띄우고(Typewriter 애니메이션 적용), 이미지는 로딩 처리(Skeleton UI).
*   **[Task 2.2] 이름 수정 모드**
    *   **작업:** AI가 지어준 이름을 인라인 Input으로 즉시 수정할 수 있는 상태 관리.
*   **[Task 2.3] Veo 비동기 대기 화면**
    *   **작업:** 5초 간격으로 `/video/status`를 폴링하며, 프로그레스 바(진행률)와 팁 텍스트("영상을 만들고 있어요!")를 번갈아 띄우는 컴포넌트 개발.

### Stage 3: 공유 / 광장 / 마무리 (Day 11~14)
*   **[Task 3.1] SNS 공유 컴포넌트 연동**
    *   **작업:** 카카오 SDK(템플릿 메시지), 트위터 공유 링크, Web Share API 연동.
*   **[Task 3.2] 도감 피드 (갤러리 뷰)**
    *   **작업:** `react-query` 또는 `SWR`을 활용한 무한 스크롤 피드 구현.
*   **[Task 3.3] 이모지 리액션 애니메이션**
    *   **작업:** 버튼 클릭 시 파티클이 터지는 CSS/Framer Motion 애니메이션 적용 및 API 호출.
