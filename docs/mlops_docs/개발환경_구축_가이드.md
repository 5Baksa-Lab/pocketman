## 4. Docker Compose 로컬 환경

모든 팀원이 `docker-compose up` 한 명령어로 전체 스택을 실행할 수 있습니다.

### docker-compose.yml

```yaml
version: "3.9"

services:

  # PostgreSQL + pgvector
  postgres:
    image: pgvector/pgvector:pg16
    ports:
      - "5432:5432"
    environment:
      POSTGRES_DB: pokeman
      POSTGRES_USER: pokeman
      POSTGRES_PASSWORD: localpassword
    volumes:
      - postgres_data:/var/lib/postgresql/data
      - ./backend/scripts/init.sql:/docker-entrypoint-initdb.d/init.sql
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U pokeman"]
      interval: 5s
      timeout: 5s
      retries: 5

  # FastAPI 백엔드
  backend:
    build: ./backend
    ports:
      - "8000:8000"
    env_file:
      - .env.local
    volumes:
      - ./backend:/app       # 코드 변경 즉시 반영 (개발용)
    depends_on:
      postgres:
        condition: service_healthy
    command: uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload

  # Next.js 프론트엔드
  frontend:
    build: ./frontend
    ports:
      - "3000:3000"
    env_file:
      - .env.local
    volumes:
      - ./frontend:/app
      - /app/node_modules
      - /app/.next
    depends_on:
      - backend

volumes:
  postgres_data:
```

### 실행 명령어

```bash
# 전체 스택 실행
docker-compose up

# 백그라운드 실행
docker-compose up -d

# 백엔드만 재빌드
docker-compose up --build backend

# 종료
docker-compose down

# 데이터 포함 완전 초기화
docker-compose down -v
```

-- pgvector 확장 설치
CREATE EXTENSION IF NOT EXISTS vector;

-- 테이블 생성 (스키마 파일에서 가져옴)
-- pokemon_master, pokemon_visual, pokemon_impression,
-- pokemon_type_affinity, pokemon_vectors,

-- 포켓몬 벡터 인덱스 (검색 속도 최적화)
CREATE INDEX ON pokemon_vectors
  USING ivfflat (feature_vector vector_cosine_ops)
  WITH (lists = 50);

## 9. API Key 발급 가이드

### Google AI Studio (Imagen 3 + Veo + Gemini Flash)

```
1. https://aistudio.google.com 접속
2. 로그인 → "Get API key" 클릭
3. "Create API key" → 프로젝트 선택
4. 발급된 키 → .env.local의 GOOGLE_API_KEY에 입력

확인:
  - Imagen 3: Gemini API에 포함 (동일 키)
  - Veo: Vertex AI 별도 확인 필요 (접근 제한 가능)
  - Gemini Flash: 동일 키로 즉시 사용 가능

무료 한도:
  - Gemini Flash: 분당 15회 (개발 시 충분)
  - Imagen 3: 분당 2회 (개발 시 Mock 이미지 사용 권장)
```

### 카카오 개발자 (공유 기능)

```
1. https://developers.kakao.com 접속
2. 애플리케이션 추가 → 앱 이름: "Pokéman"
3. 앱 키 → JavaScript 키 복사
4. 플랫폼 → Web → 사이트 도메인 등록
   개발: http://localhost:3000
   운영: https://pokeman.vercel.app (배포 후 추가)
5. .env.local의 NEXT_PUBLIC_KAKAO_JS_KEY에 입력

주의: 앱 검수 없이 바로 사용 가능 (개인 개발자 앱)
```

## 14. 개발 중 Mock 전략 (API 비용 절약)

```
개발 시:
  Imagen 3 → 사전 저장된 Mock 이미지 반환
  Veo API  → 즉시 "completed" + 샘플 영상 URL 반환
  Gemini Flash → 하드코딩된 샘플 스토리 반환

환경변수로 전환:
  USE_MOCK_AI=true   → Mock 반환
  USE_MOCK_AI=false  → 실제 API 호출

발표 당일:
  USE_MOCK_AI=false 로 전환 후 최종 테스트
```

```python
# backend/app/core/mock.py
import os

USE_MOCK = os.getenv("USE_MOCK_AI", "true").lower() == "true"

async def generate_image(prompt: str) -> str:
    if USE_MOCK:
        return "https://placehold.co/512x512/png?text=Creature"
    return await call_imagen3(prompt)
```
