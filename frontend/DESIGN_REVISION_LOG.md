# Pocketman UI/UX 리디자인 작업 기록 (Neo-Retro Pokedex)

> **작성일**: 2026-03-17
> **작업 디렉토리**: `~/Desktop/pocketman-landing-prototype`
> **원본 디렉토리**: `~/Desktop/pocketman-dev-su`

---

## 1. 작업 이유 (Why we are doing this)
원본 `pocketman-dev-su` 프로젝트는 기능 구현(얼굴 인식, AI 매칭, 크리처 생성)은 훌륭하게 되어 있었으나, UI 디자인이 지나치게 범용적인 SNS(X, Instagram 등) 형태의 베이지색 테마를 따르고 있었습니다. 이로 인해 **"나와 닮은 포켓몬을 찾는다"는 핵심 기획의 매력과 몰입감이 사용자에게 제대로 전달되지 않는 문제**가 있었습니다. 진정한 포켓몬 세계관의 감성을 살리기 위해 UI/UX 전면 개편이 필요했습니다.

## 2. 작업 목적 (Goal)
* **Neo-Retro Pokedex 컨셉 적용**: 현대적인 웹 기술(Next.js) 위에 클래식 포켓몬 도감(Pokedex)의 레트로한 디자인(Red/Blue 컬러, 게임보이 스크린, 8-bit 애니메이션)을 덧입혀 독보적인 사용자 경험을 창출합니다.
* **사용자 여정(Journey) 중심의 스토리텔링**: 딱딱한 '시스템 스펙' 설명 대신, 사용자가 직접 경험하게 될 '3단계 마법 여정(얼굴 스캔 ➔ 운명의 매칭 ➔ 새로운 진화)'으로 카피를 변경하여 호기심과 사용 욕구를 자극합니다.
* **시각적 품질 극대화**: 포켓몬 실루엣 애니메이션, 고화질 공식 일러스트 사용 등을 통해 서비스의 신뢰도와 흥미를 높입니다.

## 3. 왜 이런 방식(프로토타입 디렉토리 분리)으로 진행했는가?
원본 코드인 `pocketman-dev-su`를 직접 수정하다가 발생할 수 있는 **기능적 결함(의존성 꼬임, 백엔드 연동 오류 등)을 방지하기 위함**입니다.
1. 원본 프론트엔드 코드를 복사하여 `pocketman-landing-prototype`이라는 완벽한 **안전 샌드박스(Sandbox)**를 구축했습니다.
2. 이곳에서 백엔드 로직은 그대로 살려둔 채 UI 껍데기만 빠르고 과감하게 뜯어고칩니다 (Live Preview 활용).
3. 디자인이 우리 입맛에 완벽하게 맞춰지면, 그때 완성된 코드를 원본 디렉토리에 안전하게 덮어씌우기(Porting) 위한 전략입니다.

## 4. 지금 무엇을 하고 있었는가 (Current Status)
현재 `app/page.tsx`와 `globals.css`를 수정하여 랜딩페이지 리디자인을 90% 이상 완료한 상태입니다.
* **완료된 작업**:
  * Pokedex 레드 테마 및 BGM 자동 재생(포켓몬 사운드) 기능 추가.
  * 히어로 섹션 우측 TV 스크린에 **포켓몬 실루엣이 페이드인/아웃** 되는 애니메이션 적용 (하얀 배경 + 회색 얇은 물음표 적용 완료).
  * 하단 '광장' 영역의 예시를 밋밋한 박스에서 **고품질 공식 포켓몬 일러스트(블레이범, 스이쿤 등)**로 교체.
* **발생한 이슈 및 중단점**:
  * 아이콘 패키지(`lucide-react`) 설치 및 빌드 과정에서 Next.js 개발 서버 캐시(`node_modules/react`)가 엉키는 현상이 발생했습니다. 
  * 꼬인 `node_modules` 폴더를 강제 삭제하고 재설치하여 에러를 복구하려던 중 폴더 Lock 현상으로 인해 작업 기록을 먼저 남기고 있습니다.
